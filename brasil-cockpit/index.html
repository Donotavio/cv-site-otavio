05

Pipeline do Cockpit

Medallion architecture sobre 4 fontes independentes — SGS, PTAX, SIDRA e mercado — reconciliadas num painel de status.

Bronze[ L1 ]

SGS + PTAX + SIDRA + yfinance → Parquet

SGS (Banco Central) para séries mensais/diárias, PTAX via OData para câmbio (USD/EUR), SIDRA/IBGE para a PNAD Contínua (desocupação) e yfinance para o Ibovespa — cada fonte preservada em Parquet particionado por data de coleta.

PythonBACEN SGSPTAXIBGE SIDRAyfinance
Silver[ L2 ]

DuckDB normaliza séries → schema unificado

DuckDB alinha todas as séries num schema único (kpi_id, data_referencia, valor), com conversão de unidade onde preciso (US$ mi → bi para a balança). Rebuild completo a cada execução — sem estado incremental.

DuckDBParquet
Gold[ L3 ]

Status + delta + trend → cockpit.json

compute_status / compute_trendavaliam cada KPI contra catalog.TARGETS (meta ± banda + direction), produzem status/delta/trend e exportamcockpit.json + historico.json, consumidos direto pelo site em runtime.

DuckDBJSONGitHub Pages
GitHub Actions

Coleta diária, mensal e trimestral

PTAX + mercado disparam todo dia às 13h UTC; séries mensais do SGS rodam no fechamento do mês; IBGE (PNAD) roda trimestral. Fail-soft: se uma série dá erro, ela é pulada e o painel segue — um KPI a menos, nunca uma página quebrada.

update-profile.yml
  cron: diário · 13h UTC
  ├── collect_ptax.py
  ├── collect_market.py
  │     └── Bronze Parquet
  └── silver_macro.py → gold_cockpit.py
        └── cockpit.json + historico.json

Metodologia & limitações

Transparência total sobre o que esse painel mede — e o que ele ainda não consegue medir.

  • CAGED não entra

    Os códigos SGS 28763/4/5 (admitidos/desligados/saldo) são cumulativos — estoque/YTD, não fluxo mensal. Um rolling 12m sobre eles seria inválido. O emprego no cockpit vem da PNAD Contínua (trimestral, tabela 4099 do SIDRA), que é a série oficial de desocupação.

  • PIB, PIM, PMC e PMS ficaram de fora

    dado indisponível

    As tabelas SIDRA de atividade (PIB, produção industrial, comércio e serviços) estão encerradas ou vazias no endpoint atual — só a PNAD (4099) está estável. Serão reativadas quando os códigos das tabelas-base atuais do SIDRA forem identificados.

  • Status é composto, não faixa fixa

    Verde / amarelo / vermelho vêm de meta ± banda + directionpor KPI — calculado em catalog.compute_status. Não é uma faixa fixa igual para todos: cada indicador tem sua própria meta, banda e direção (alto_melhor ou baixo_melhor).

  • reservas: endpoint SGS intermitente

    O endpoint de reservas internacionais no SGS às vezes times out. Fail-soft: se a coleta falha, o KPI fica ausente sem quebrar o painel — ele simplesmente não aparece na categoria externo até a próxima execução bem-sucedida.

fontes: BACEN SGS · PTAX OData · IBGE SIDRA · yfinance · coleta diária/mensal/trimestral via GitHub Actions
