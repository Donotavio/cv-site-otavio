02

Arquitetura do Pipeline

Medallion architecture sobre dados abertos do Banco Central do Brasil.

Bronze[ L1 ]

BACEN API → Parquet Raw

Coleta diária via python-bcb. Dados brutos preservados em Parquet particionado por data — imutável, auditável, versionado.

Pythonpython-bcbParquet
Silver[ L2 ]

Limpeza e Tipagem

Normalização de nomes, inferência de tipos, deduplicação e enriquecimento de datas. Schema enforced via DuckDB.

DuckDBPandasParquet
Gold[ L3 ]

Agregações Analíticas

Visões pré-computadas para dashboards: série temporal diária, crescimento acumulado, ticket médio por período.

DuckDB WASMD3.jsChat demo
GitHub Actions

Orquestração automatizada

Pipeline executado diariamente via GitHub Actions: coleta → transforma → publica. Zero infraestrutura — serverless completo no GitHub. Retry automático em falha, alertas por issue e histórico rastreável.

schedule: cron

03

Insights dos Dados

Padrões descobertos ao analisar 4+ anos de histórico do PIX.

Nov/202001

Crescimento histórico

Dia 1 do PIX: 2.345 transações. Hoje: 258M+/dia. Um crescimento de 106.000× em menos de 5 anos — o mais rápido da história dos sistemas de pagamento.

2.345→258M+
B2B02

Evolução do ticket médio

Ticket médio saiu de R$90 para R$700+ nos picos. O PIX virou instrumento B2B — empresas movem boladas fora do horário bancário, algo impensável antes de novembro/2020.

R$ 90→R$ 700+
SLA03

Resiliência do sistema

Disponibilidade SLA: 99,9%+ histórico. Um dos sistemas de pagamento mais resilientes do mundo, operando 24/7/365 com tempo de resposta médio < 10s.

99,9%+uptime histórico
