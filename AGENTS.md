# Repository Guidelines

## Project Structure & Module Organization

- `_config.yml`: Jekyll configuration for GitHub Pages.
- `_layouts/` and `_includes/`: page templates and shared HTML partials.
- `index.html`: entry page with front matter that selects a layout.
- `assets/css/`, `assets/js/`, `assets/img/`, `assets/cv/`: static front-end assets.
- `assets/i18n/`: translation dictionaries (`pt-BR`, `en-US`, `es-ES`) used by `data-i18n-key`.
- `assets/data/`: JSON data for profile, GitHub, and LinkedIn; usually generated by scripts.
- `scripts/`: Python utilities for data fetching/translation.
- `dist/`: generated build output (Astro); do not edit by hand.
- `src/`: Astro components, layouts, styles e scripts de motion.
- `public/assets/`: symlinks para `assets/*` (dados, i18n, img, fonts, cv, js).

## Build, Test, and Development Commands

- `npm install`: instala dependências (Astro, GSAP, Lenis).
- `npm run dev`: servidor de desenvolvimento Astro.
- `npm run build`: build estático em `dist/`.
- `npm run preview`: serve o `dist/` localmente para validação.
- `python3 scripts/fetch_github_data.py`: updates `assets/data/github_activity.json` (requires `GITHUB_USERNAME` and `GITHUB_TOKEN`).
- `python3 scripts/fetch_linkedin_data.py`: updates LinkedIn JSON files (requires `LINKEDIN_PROFILE_URL` and `LINKEDIN_SESSION_COOKIE`).
- `python3 scripts/translate_projects.py`: regenerate translated project strings when needed.

Deploy: push para `main` dispara `.github/workflows/build-and-deploy.yml`
(Astro build → GitHub Pages via Actions). O pipeline de dados
(`update-profile.yml`) continua atualizando os JSON em `assets/data/`.

## Coding Style & Naming Conventions

- Stack: Astro (HTML estático) + CSS vanilla + TypeScript. Motion via GSAP + Lenis
  (únicas libs externas permitidas). Sem outros frameworks de UI.
- Tokens de design são lei: usar CSS custom properties de `src/styles/tokens.css`,
  nunca hex hardcoded. Ver `DESIGN.md` e `.opencode/skills/design-system.md`.
- Indent with 2 spaces in HTML, CSS, and JS.
- Use kebab-case for CSS classes and data attributes (e.g., `hero-title`, `data-i18n-key`).
- Keep translations in `assets/i18n/*.json` and reference keys in markup.
- Conteúdo dinâmico (timeline, projetos, stats, blog) é buscado via `fetch` dos
  `assets/data/*.json` em runtime — o pipeline Python não muda.

## Testing Guidelines

- No automated tests are configured.
- Manually verify key flows: language switcher, timeline interactions, and layout on desktop/mobile.

## Commit & Pull Request Guidelines

- Git history is not available in this checkout; use short imperative messages (e.g., "Add timeline animation").
- PRs should include a brief summary, screenshots for UI changes, and any data/script updates.

## Security & Configuration Tips

- Store tokens and cookies in environment variables or GitHub Actions secrets.
- Never commit `GITHUB_TOKEN` or LinkedIn session cookies.
