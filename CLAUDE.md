## Project Overview

Personal portfolio site for a Data Engineering Manager, built with Jekyll and deployed to GitHub Pages. Uses plain HTML/CSS/JS (no frameworks or external JS libraries). Supports three languages: pt-BR, en-US, es-ES.

Live at: https://donotavio.github.io/cv-site-otavio/

## Build & Development Commands

```bash
bundle install                              # Install Jekyll dependencies
bundle exec jekyll serve --livereload       # Local dev server at localhost:4000
bundle exec jekyll build                    # Build static site into _site/
```

### Data Pipeline Scripts (Python 3.11)

```bash
pip install requests beautifulsoup4         # Python dependencies
python3 scripts/fetch_github_data.py        # Update github_activity.json (needs GITHUB_USERNAME, GITHUB_TOKEN)
python3 scripts/fetch_linkedin_data_enhanced.py  # Update LinkedIn JSON (needs LINKEDIN_PROFILE_URL, LINKEDIN_SESSION_COOKIE)
python3 scripts/translate_projects.py       # Regenerate translated project strings
python3 scripts/build_profile_data.py       # Build consolidated profile.json
```

There are no automated tests. Verify changes manually: language switcher, timeline interactions, responsive layout.

## Architecture

### Jekyll Layer
- `_config.yml` — Jekyll config (baseurl: `/cv-site-otavio`, lang: `pt-BR`)
- `_layouts/default.html` — Base HTML shell; `_layouts/home.html` — Main page layout
- `_includes/` — Reusable partials: header, footer, navbar, timeline, language_switcher, roi-calculator
- `index.html` — Entry point, just front matter selecting `home` layout

### Frontend JS Modules (all vanilla, loaded as separate `<script>` tags)
- `main.js` — Orchestrator: fetches JSON data, renders timeline/content, coordinates other modules
- `i18n.js` — Internationalization engine: loads `assets/i18n/*.json`, handles `data-i18n-key` attribute translation, language switching without reload
- `content.js` — Dynamic content rendering from JSON data
- `timeline.js` — Interactive career timeline component
- `animations.js` — Scroll-triggered animations and transitions
- `hero-background.js` — Canvas-based animated background
- `scroll.js` — Scroll effects and navigation
- `roi-calculator.js` — ROI calculator widget
- `theme.js` — Theme handling

### Data Layer (`assets/data/`)
JSON files that drive all content. Most are auto-generated by Python scripts via GitHub Actions (daily cron at 06:00 UTC):
- `profile.json` — Consolidated profile data (built by `build_profile_data.py`)
- `github_activity.json` — GitHub stats and repos
- `linkedin_profile.json`, `linkedin_recommendations.json` — LinkedIn data
- `projects_extended.json`, `tech_stack.json`, `blog_articles.json` — Content data

### i18n (`assets/i18n/`)
Translation dictionaries: `pt-BR.json`, `en-US.json`, `es-ES.json`. Referenced in HTML via `data-i18n-key` attributes. The JS `i18n.js` module swaps text at runtime.

## Coding Conventions

- Plain HTML/CSS/JS only — no frameworks or bundlers
- 2-space indentation in HTML, CSS, and JS
- kebab-case for CSS classes and data attributes (e.g., `hero-title`, `data-i18n-key`)
- Commit messages use Conventional Commits format (e.g., `feat:`, `fix:`, `chore:`)
- CSS variables defined in `:root` in `assets/css/main.css` for theming

## CI/CD

GitHub Actions workflow (`.github/workflows/update-profile.yml`) runs daily to fetch fresh GitHub/LinkedIn data, translate projects, build profile JSON, and auto-commit changes to `assets/data/` and `assets/i18n/`.
